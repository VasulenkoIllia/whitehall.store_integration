# Що зроблено понад початкове ТЗ

Нижче — функціонал, якого **немає** в CLIENT_TZ.md, але він уже реалізований.

## Доступ і UX
- Режим **read‑only** на `/` (тільки перегляд без змін).
- Повна адмінка на `/admin` з розбиттям на вкладки: статус, постачальники, merged, finalize, Horoshop, API preview, порівняння, логи, ручне керування, планування.
- Пошук, фільтри, сортування, пагінація у таблицях.
- Модальні вікна для редагування постачальників/джерел.

## Джерела і мапінг
- **Кілька джерел на одного постачальника** (із власними назвами).
- **Мапінг на рівні джерела** (source_id), а не лише постачальника.
- Мапінг із **статичними значеннями** та можливістю обрати “немає значення”.
- Підтримка “без заголовків” для таблиць.

## Ціноутворення (додаткові опції)
- Мінімальна націнка як **налаштування**: увімкнути/вимкнути + окрема сума.
- Масова зміна % націнки для кількох постачальників.
- Ручна ціна з можливістю вимкнути та повернутись до автоматичного розрахунку.

## Horoshop
- **Повна синхронізація** Horoshop в локальне “дзеркало”.
- **API preview** — окремий набір даних для Horoshop з переглядом в UI.
- Вкладка **порівняння** (Final vs Horoshop) для контролю того, що ще не на сайті.
- Обробка rate‑limit та оновлення токену при імпорті/синхронізації.

## Контроль виконання
- Плановий процес як **єдиний pipeline** (import → finalize → export → Horoshop import).
- Блокування паралельних job, **скасування** та **таймаут** завислих job.
- Детальні логи з відкриттям помилки та розгорнутими даними.

## Сповіщення
- Надсилання **помилок у Telegram‑бота**.

## Інфраструктура
- Docker Compose з Postgres і healthcheck‑ами.
- Внутрішній endpoint `/health`.
